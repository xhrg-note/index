## dubbo


基于dubbo的服务话，一般是这样的。公司的内部环境分为dev，fat，uat，pro四个环境。那么我们可以搭建四个环境集群的zookeeper作为全公司的注册中心。

基于dubbo的服务话，有三个点需要注意

#### 服务提供者

一般服务提供者需要维护服务提供者的服务注册到注册中心，还要维护服务提供者的sdk。


#### 服务消费者

服务消费者拿到服务提供者的sdk，然后依赖到自己的项目中，放上注解。


因为服务提供者和服务消费者在同一个环境，用的是同一套zookeeper，那么消费者的sdk可以拿到提供者的地址进行调用。dubbo有一个很大的好处是，提供的sdk，只有接口，无需有任何注解之类的特殊处理。
