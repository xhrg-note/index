## 分层

分层的常见原则

* 上层依赖下层，但是下层不依赖上层。平级之间相互依赖视情况而定。
* 分层需要包含开发的方方面面，而不是局部概括。


一个比较好的分层结构

```
/用户接口层
  /webapi
  /身份认证
  /dto转换
/应用层
  /引用服务
  /时间监听
  /消息监听
  /消息监听
/领域层
  /聚合
  /领域服务
/基础设施层
  /数据库
  /消息队列
  /缓存
  /远程接口
  /搜索引擎
  /工具类
  /基础服务


从上到下，依次是，用户接口->应用层->领域层->基础设施层，而基础设施层可以被每一层访问
```
